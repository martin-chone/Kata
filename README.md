## 🧪 Approche TDD

Chaque kata suit le cycle **Rouge → Vert → Refactorisation** :
1. Écrire un test qui échoue
2. Écrire le code minimal pour le faire passer
3. Améliorer le code (refactorisation)

Les tests sont écrits avec **xUnit**.
